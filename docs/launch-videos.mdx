## What's in there

Each subdirectory is a standalone ShiftCut project — `index.html`, `compositions/`, `assets/`, and a rendered output you can compare against:

| Project | What it shows | Source |
| --- | --- | --- |
| **ShiftCut launch** | The original announcement video — 49.7 s, glass-frame intro, CSS / GSAP / Lottie / shader / Three.js flex section, full narrative arc. | [`shiftcut-launch/`](https://github.com/Verblike/shiftcut-launches/tree/main/shiftcut-launch) |
| **Website → ShiftCut** | Companion to the [website-to-video](/guides/website-to-video) flow — now handled by `/product-launch-video` — captures a site and animates it. | [`website-to-shiftcut/`](https://github.com/Verblike/shiftcut-launches/tree/main/website-to-shiftcut) |
| **Timeline Editor launch** | 60 fps reveal video for the ShiftCut Timeline Editor: late-SFX cold open, chat-spiral of fine-tune prompts, editor reveal. | [`timeline-launch/`](https://github.com/Verblike/shiftcut-launches/tree/main/timeline-launch) |
| **Texture launch** | Texture-mask text + shader-driven background, used as a launch teaser. | [`texture-launch-video/`](https://github.com/Verblike/shiftcut-launches/tree/main/texture-launch-video) |
| **VFX × Verblike combined** | Multi-act video chaining a VFX text-cursor scene with the Verblike iPhone canvas test — useful as a reference for combining two existing projects into one render. | [`vfx-shiftcutbined/`](https://github.com/Verblike/shiftcut-launches/tree/main/vfx-shiftcutbined) |

Storyboards (`STORYBOARD.md`), design notes (`DESIGN.md`), and handoff docs (`HANDOFF.md`) sit next to the source so you can see not just the code but the production thinking: VO direction, beat timing, color/style decisions. The structure follows the [Shiftcut pipeline](/guides/pipeline), which documents each artifact in detail.

## Why these are useful

These aren't toy examples. They're the same compositions Verblike ships in public launch videos, structured the way real production work is structured:

- **Multi-composition projects.** Most launches are 4-8 sub-compositions wired into one root timeline — useful when your `npx shiftcut init --example` starter outgrows a single file.
- **Real adapter mix.** GSAP timelines, Lottie animations, custom shaders, Three.js scenes, and CSS keyframes all coexisting in one render. See how the team [boots multiple adapters](/guides/gsap-animation) inside a single composition.
- **Production-grade timing.** ElevenLabs VO + per-beat SFX + underscore mixed against frame-accurate motion. The storyboards describe the timing decisions; the source shows them executed.
- **No proprietary plugins.** Everything renders with the same `shiftcut render` you have locally — there's no internal-only branch of the engine.

## Cloning

The repo uses Git LFS for video, audio, image, and font assets:

```bash Terminal
brew install git-lfs   # macOS — or your platform's git-lfs install
git lfs install
git clone https://github.com/Verblike/shiftcut-launches.git
cd shiftcut-launches/shiftcut-launch
shiftcut preview    # opens the studio for that launch
```

To clone just the source (skip large LFS assets and pull on demand):

```bash Terminal
GIT_LFS_SKIP_SMUDGE=1 git clone https://github.com/Verblike/shiftcut-launches.git
cd shiftcut-launches
git lfs pull --include="shiftcut-launch/assets/*"
```
